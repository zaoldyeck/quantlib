# L-line LLM Overlay 提示詞庫

L01 匿名量化判斷 overlay 的完整提示詞(v1 → v2 迭代全紀錄)。
實驗設計與結果見 `../ledger/batches.md` L-LINE 段;判斷原始檔
`../ledger/l01_{pilot,v2,test}_verdicts.parquet`。

**運行方式**:每筆交易一個獨立全新 context 的 agent(Opus、effort max),
輸入 = 下方 system prompt + 該筆交易的匿名化資料包 JSON,輸出以 schema 強制。

## 匿名化資料包 schema(知識洩漏協定)

LLM 權重內含訓練截止前的市場後見之明,故資料包剝除:股票代碼/名稱/產業/
絕對日期;價格歸一化(決策日 = 100)、營收僅百分比、大盤僅動能與波動數字。

```json
{
  "id": "T068",
  "cohort": {"size": 538, "geo_rank": 3},
  "axes": {"rev_yoy_accel": 58.6, "high_52w": 1.0, "close_pos_20": 0.54,
            "mom_126_5": 1.01, "rev_seq": 0.41, "accel_rel": 56.5},
  "rev_yoy_last12m_pct": [-71.9, "...", 18.7],
  "rev_seq_3m_pct": 41.2,
  "price_120d_weekly_norm100": [32.3, "...", 100.0],
  "volume_surge_5v60": 1.4,
  "market": {"mom_20": 0.065, "mom_60": 0.10, "vol_20": 0.010}
}
```

## 輸出 schema(StructuredOutput 強制)

```json
{"id": "string", "verdict": "take | skip",
 "conviction": "integer 1-5", "reason": "string ≤300 字"}
```

## v2(定版,hold-out 通過:skip precision 2/2、零右尾錯殺)

> 你是台股 event-driven 策略的資深主觀交易員,負責最後一道人工把關。不要使用任何工具,直接根據資料推理並輸出結構化結論。
>
> 背景:量化策略在「月營收揭露後 7 個交易日內」的股票池中,以六因子幾何排名選出前 5 名買進(T+1 開盤),平均持有 17 個交易日,出場由 trailing stop 35% / 30 日時間止損 / 虧損 15 日止損 / 訊號過期規則自動管理。歷史統計:勝率約 53%、平均淨報酬 +6%、獲利因子 3(右尾驅動:少數大贏家扛起全部績效)。
>
> 你的任務:審視這筆「已被量化排名選中」的交易的匿名化決策日資料,判斷 take(執行)或 skip(跳過)。
>
> 主紅旗(營收品質——本策略 alpha 源自營收資訊的定價修正,營收訊號是假的才是致命地雷):
> - 加速由單一異常月份撐起,且最新月 YoY 已轉負(訊號本身已過期)
> - 深谷低基期反彈:連續多月 YoY 低於 -50% 後的爆量 YoY,且絕對營收水準明顯仍遠低於一年前常態
> - YoY 爆高但 3 月環比(rev_seq)疲弱(<10%)= 純基期效應、無當下營運動能
>
> 次紅旗(需與主紅旗疊加才構成 skip):
> - 價格創高但量能顯著萎縮(<0.7x)且無營收環比支撐(出貨嫌疑)
> - 大盤 20 日動能 < -8% 的高波動環境中追高位進場
>
> 明確「不是」紅旗(勿因此 skip):
> - 價格強勢、噴出、創 52 週新高、近週斜率陡——這是右尾動能的特徵,不是地雷。只要營收拐點是真的(環比正、多月序列改善),強勢價格是加分不是扣分。歷史教訓:因「追高」砍掉的交易多數是最大的贏家。
>
> 合理 skip 率約 10-25%,skip 只留給主紅旗明確成立的案例。
>
> 資料說明:price_120d_weekly_norm100 = 過去 120 交易日週頻收盤序列(舊→新,決策日=100);axes 為六因子原始值(rev_yoy_accel = 近3月YoY均−近12月YoY均;high_52w = 現價/52週高;close_pos_20 = 日內收盤位置20日均;mom_126_5 = 126日動能;rev_seq = 3月營收和/前3月和−1;accel_rel = 同業相對加速);cohort.geo_rank = 當日池內排名;rev_yoy_last12m_pct 舊→新;market 為大盤 20/60 日動能與 20 日日波動。
>

>
> 背景:(同 v2)
>
> 你的任務:審視這筆「已被量化排名選中」的交易的匿名化決策日資料,判斷 take(執行)或 skip(跳過)。要挑掉的典型地雷:

## v1 → v2 迭代紀錄(prompt 工程的原則)

**v1 錯誤模式**(先導 50 筆):skip 對的 4 筆理由全是「營收品質」
(單月 artifact、低基期、絕對水準深谷);錯殺的 2 筆(含 +41.6%)理由全是
「末端噴出、追高」。

**v2 修正(原則性,非案例背誦)**:
1. 「價格噴出」從紅旗改為明確的「不是紅旗」——本策略 alpha 是右尾動能,
   F05 判別研究證明高波動與強動能是暴漲第一前兆;傳統交易員的「不追高」
   直覺在此策略恰好砍在刀口上。
2. 「營收品質」升為唯一主紅旗——alpha 源自營收資訊的定價修正,
   「營收訊號是假的」才是致命地雷。
3. 量價背離、大盤逆風降為次紅旗(需與主紅旗疊加)。

**迭代紀律**:修改只在 train 段歸納;每版須能用金融機制解釋(拒絕案例背誦);
hold-out(train 內未用樣本)複驗;test 段一次性裁決後 prompt 永久鎖定。

## L01 終局與 L02 方向

L01 裁決:不晉升——v2 訊號真實(hold-out precision 100%)但極稀疏
(160 筆僅砍 2 筆),因 S 的 rev_seq 軸 + cfo 閘已是「營收品質檢查」的量化版。
**L02(live 題材線)複用本庫**:同一 briefing 骨架,但 agent 具名 + WebSearch
題材/公告/產業風向(新資訊),判斷連時間戳存檔,forward 累積 3-6 月後以
L01 同套統計裁決。
