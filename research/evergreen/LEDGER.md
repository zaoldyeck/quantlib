# Evergreen — Agent 泛化策展 × 量化引擎(program 章程與 trials ledger)

**立案**:2026-07-10|**使用者指令**:Serenity 方法論的泛化——Agent 要能在
任何產業任何時機找出標股,系統長青;持股數不限,一切以回測績效決定;
Agent 一律 Opus 4.8 effort max;token 預算節制。

## 系統設計(四階段)

```
Phase 1 歸因    train 窗暴漲股 → Agent 分析暴漲原因(可用題材)→ 結構化歸因
Phase 2 蒸餾    歸因報告 → 「暴漲哲學」(泛化判斷框架)→ 寫成標記 prompt
Phase 3 標記    逐月:量化 pre-screen 縮池 → Agent 依哲學標記訊號股
                → 時變池 registry(泛化版 thesis registry)
Phase 4 疊加    Agent 池 × 量化引擎(進出場/計分/風控)→ train 調參
                → OOS 一次性驗證 + 對照實驗
```

## 時間窗合約(不可變)

| 窗 | 範圍 | 用途 |
|---|---|---|
| train | 2022-07-01 → 2025-06-30(3 年,M01 最優窗) | 歸因、哲學蒸餾、標記、量化調參 |
| **OOS** | **2025-07-01 → 2026-07-09(1 年)** | **一次性驗證,任何層不得回頭調** |

**哲學蒸餾只准看 train 窗的暴漲事件**(OOS 的果不得進 prompt)。

## 知識洩漏防禦(繼承 Serenity 三道防禦 + 梯度聲明)

1. **證據日期合約**:Phase 3 每個標記必附 evidence_date(最早可公開查證的
   證據時點)+ evidence_type + reason;訊號生效日 = max(標記月, evidence_date)。
2. **Activation-lag 壓力窗**:全部標記整體延後 90 / 180 天重跑 Phase 4;
   lag 後崩壞 = 後見之明主導,系統作廢。
3. **對照實驗**:(a) 同 pre-screen 無 Agent 的純機械對照(隔離 Agent 貢獻);
   (b) 池內置換檢定(隨機選股保留閘門);(c) Agent 池的事後命中率披露。
4. **污染梯度誠實聲明**:Opus 4.8 權重含 ~2025 中前的市場記憶——train 窗
   標記受污染(靠 1-3 控制上限),OOS 窗後段(cutoff 後)漸趨乾淨,
   live(2026-07 起)全乾淨。三段分別報告。

## EV3 設計修正(recall 校準,2026-07-10)

**發現**:量化綜合排序對未來暴漲事件的 recall 極低(top-80/120/160/240 =
8/12/17/23%)——暴漲前的股票多數「看起來平庸」(F05 結論再現),截斷式
pre-screen 會先砍掉 77% 的可及事件。
**修正**:放棄截斷,改**全池壓縮表**(每檔一行:代碼/名稱/產業/yoy3/accel/
h52/mom60/vsurge,~1,100 檔 ≈ 33K token/月)→ recall 100%,Agent 直接
掃全市場標記 5-15 檔。成本 49 月 × ~40K ≈ 200 萬 token(預算內)。
Agent 價值定位:在平庸數字中辨識結構意義(利基地位/產業鏈位置),
此為量化排序在資訊論上做不到之事。

# EV11 — 系統形態修正:事件驅動日級標記(2026-07-10;使用者糾正)

**使用者糾正(原話重點)**:「實際交易系統每天跑,每天掃消息面,題材是日頻的;
標記 = 這支股票在某個特定日期出現訊號,某天之後才入池,之後才是量化條件」——
EV3 把此設計折衷成「月初出勤快照名單」= 形態錯誤(粒度月級、入池日被
量化到月初、live 運作形態偏離)。

**EV11 正確形態**:
1. **事件流**(本地,PIT):每日偵測顯著事件——創高(首破 120 日高 × 貼 52 週高
   × 帶量)、爆量急漲(vx>5 & +5%)、營收爆發(yoy3>40 或 acc>30,發布日)——
   每 (股票, 事件類) 60 交易日冷卻;月均 277 事件(漏掉的弱事件會被後續更強
   事件補上,無一次性截斷的 recall 問題)。
2. **Agent 日頻判斷**(回測 = 每月一個 agent 讀整月事件流逐日審):構成訊號者
   輸出 (code, **signal_date = 事件日**, conviction, type);判斷某日事件僅用
   ≤ 該日資訊。
3. **池**:signal_date + 1 起入池(v3 席位引擎照跑)。
4. Live 形態 = 每日收盤後掃當日事件 + 消息面 → 當日判斷 → 次日生效
   (回測與 live 形態一致,僅 agent 出勤打包粒度不同)。

試點:3 個月(2023-02/2024-03/2025-04)驗管線與標記行為 → 呈報全量成本。
