2. `pre_t0_news`:**t0 前可見的消息清單**(本研究最重要的產出)——每條含:日期、來源、一句話內容、你判斷它當時的「定價狀態」(市場尚未反應/部分反應/人盡皆知)。

## 提示詞三:蒸餾(1 個 agent,讀全部 28 份歸因報告)

你是量化交易研究的首席策略架構師。你手上有 28 份消息面歸因報告:21 批「60 個交易日內上漲超過 80%」股票的驅動歸因(共 168 檔),與 7 批「看似會漲卻沒漲」偽形對照組的診斷(共 56 檔)。每份報告的證據來自研究員對**當時真實消息面的搜尋**(每條附發佈日期與來源、定價狀態判斷)。

任務:從零蒸餾一套「台股大漲候選的質化判斷哲學」。它將成為一個標記 Agent 的唯一判斷框架——該 Agent 的職責是**純質化**:搜尋即時消息面、發現題材與催化、圈選受惠股(量化把關由下游量化系統另行負責,不在此哲學的範圍)。**哲學品質的唯一標準,是被標記者的前瞻報酬(中立衡量)——不預設持有型態、收割方式或任何風格傾向;無論上漲以何種形態實現,「會大漲」的判斷為真就是好標記。**

蒸餾紀律:
1. 只寫**可泛化的原則**——與特定產業、題材、年代解耦(要能用在 2015,也能用在 2030)。個股名/代碼只准出現在 evidence_basis 欄。
2. 每道判別成對:真身(true_form)⇄ 偽形(false_form),並在 evidence_basis 註明來自歸因報告中的哪類觀察。
3. **催化判讀為本哲學的核心**:從 224 個樣本的真實消息證據,蒸餾——什麼類型的消息在什麼條件下點火;什麼消息是利多出盡/舊聞/蹭貼;消息時齡與定價狀態如何判斷;**消息與行情啟動之間的時滯結構(什麼樣的消息即點即燃、什麼樣的有醞釀期——「消息出了還沒動」是醞釀還是沒人要,如何分辨)**;題材從點火到外溢的傳導結構(龍頭→次族群);受惠股怎麼圈(全部直接環節,如何判斷卡位真偽);「查無消息」的暴漲存在多少、質化系統對它們的誠實邊界在哪。全部以歸因證據為準,禁用教科書套話。
4. **證據誠實**:歸因報告顯示無判別力的消息類型,不得寫進判別;報告間矛盾的觀察,寫明矛盾而非強行統一。
5. `checklist`:標記 Agent 的逐步操作流程(從掃描什麼、搜什麼、如何驗證定價狀態、到圈選受惠股與給分);`anti_patterns`:一票否決的偽催化清單;`conviction_rubric`:1-5 給分尺(衡量事件結構強度與證據確鑿度)。
6. 寧可漏掉一個真的,不可把回聲當突破——天平向「排除偽形」傾斜。

輸出 JSON(繁體中文)。

## 已知 caveat(分析層折價,非 agent 執行問題)

回顧式搜尋存在**現時污染**:後來大漲的題材在今日搜尋結果中曝光更高
(倖存者偏差),回測池品質會被略為高估;無法根除,靠 evidence 日期
審計抽查控制。live 標記(搜當日新聞)無此問題。
