# A-ex_right_dividend — 解析正確性稽核

**一句話結論**:舊制檔案（2003–2024/06 的 twse、2008 起大部分的 tpex）的除權息資料**完全可信**
——每一欄都對、單位對、日期對、正負號對、編碼沒走樣，我另寫一支不呼叫官方解析器的獨立解析器逐格對過，
還拿收盤價去跟 `daily_quote` 的真實前一日收盤交叉驗證（8,539 筆裡 99.6% 落在 1% 內）。
**但 2024-07 之後的新制（MOPS 月檔）資料只對了一半**:現金股利（配息）還原得回來，
但**純配股（除權）事件被存成整列 0**，害下游的還原價（`prices.py`）在那些除權日**還原不回去、留下假跌**
——2024-07 至今共 **59 檔**中鏢，含元大金、富邦金、國巨、凱基金、保瑞、信驊等大型股。這件事程式碼裡
已有 TODO 標記（`FC1-parse`）知道要補,但**還沒修**。

## 能信的部分（負結果照樣落盤）

### 1. 舊制解析逐格對帳:零差異
獨立解析器（`01_reparse_parity.py`,自己讀 Big5-HKSCS、自己切 CSV，**不碰 `TradingReader`**）
重解了跨時代抽樣的檔案,和 PG 逐欄比對:

| 檔案 | 涵蓋 | 筆數 | 數值不符 | 只在raw |
|---|---|---|---|---|
| twse/2020_7_15 | 2020-07-15 | 21 | 0 | 0 |
| tpex/2020_7_15 | 2020-07-15 | 11 | 0 | 0 |
| **tpex/2020_7_10（22欄大批量檔）** | **2008-01-10 ~ 2020-07-10** | **6,156** | **0** | **0** |
| twse/2022_4_27 | 2022-04 | 11(+7 ETF*) | 0 | — |
| tpex/2023_8_19 | 2023-07~08 | 164 | 0 | 0 |
| tpex/2026_1_21（舊制範圍檔） | 2025-12~2026-01 | 108 | 0 | 0 |

八個價量欄（除權息前收盤、參考價、權值+息值、漲停、跌停、開盤競價基準、減除股利參考價、權/息別）
全部逐格相符。那個 6,156 筆、橫跨 12 年（2008–2020）的大批量檔零差異,直接證明:
（a）解析器用的是**檔案內容的日期**而非檔名日期（檔名叫 2020_7_10,內容卻是 2008–2020,日期全對）;
（b）那個唯一一個 22 欄變體檔（比別人多一欄「員工紅利轉增資」插在第 15 欄）**沒有造成欄位錯位**
——因為解析器只讀到第 12 欄,多出來的欄在後面,咬不到。

\* 那 7 筆「只在 raw」是我的解析器沒去掉 TWSE 的 Excel 文字防護殼 `="0052"`,官方解析器的
`QuantlibCSVReader` 會把整行的 `=` 拿掉,所以 DB 裡是乾淨的 `0052`（實查:0052 現金股利 10.19、
00714、00717、01003T 全部入庫正確）。這反而證明解析器的 `=` 清洗有效、沒漏 ETF。

### 2. 欄位語意交叉驗證:對得上真實市價
把 DB 存的「除權息前收盤價」拿去比對 `daily_quote` 裡**除權日前一交易日的真實收盤**（獨立資料源）:

- tpex 舊制 4,291 筆:99.6%（4,274 筆）落在 1% 內,超過 5% 的**0 筆**。
- twse 舊制 4,248 筆:99.6%（4,230 筆）落在 1% 內,超過 5% 的**只有 1 筆**（和康生 1783 @ 2021-08-19,
  65.2 vs 69.6,交易所公告的前收盤與實際收盤本身有落差,是來源特性不是解析錯位;若是錯位會是幾千筆全歪而非 1 筆）。

這證明「除權息前收盤價」這一欄真的裝的是收盤價、沒跟隔壁欄互換。

### 3. 其它都排除了
- **單位**:價格都是「元」,和 daily_quote 同量級;沒有元/千元/張/股混用。
- **正負號**:全部 ≥ 0,無錯誤負值。
- **民國轉西元**:twse 用「109年07月15日」、tpex 用「109/07/15」,兩種都正確 +1911。
- **編碼**:Big5-HKSCS 解碼中文欄名/股名無亂碼;中文只出現在會被過濾的標頭與股名。
- **欄位數漂移**:掃過全部 237 檔,twse 一律 16 欄、tpex 102 檔 21 欄 + 1 檔 22 欄(已證無害);
  MOPS 月檔一律 19 欄且內嵌逗號的欄位(參加分派總股數 `"35,782,800"`)在第 15 欄,解析器只讀到第 10 欄,
  CSV 解析器又正確處理引號,咬不到。
- **檔名 vs 內容日期**:解析器一律用內容日期,tpex「日檔」其實是範圍檔(檔名取區間結尾日),不影響正確性。

## 不能全信的部分

### BUG 1 —（2024-07 起）純配股除權事件被存成整列 0,下游還原不回來
**現象**:2024-06 之後 twse 舊制端點停擺,改抓 MOPS t108sb27 月檔;MOPS 檔**沒有**收盤價/參考價/漲跌停
這些欄,解析器 `parseMopsRows` 把它們全部寫 0(這部分是來源限制,情有可原)。**但問題在配股**:一筆純配股
（除權）事件,解析器產出的列是 `cash_dividend=0` 且所有價格欄=0。而 MOPS 檔其實**有**「股票股利(元/股)」
這兩欄（盈餘轉增資、資本公積轉增資,第 4、5 欄）——解析器**讀了、卻只拿來當『>0 就發一列』的布林判斷,
數值本身丟掉**。於是下游 `prices.py` 算還原因子時:①優先用參考價/前收盤(都 0,不成立)②退而用現金股利
(也 0,不成立)③ 兩者皆無 → 因子 = null → 被濾掉。結果那天股價因配股自然跳水,卻沒有因子還原它 → **假跌**。

**衝擊面**:2024-07 至 2026-07 共 **59 檔**中鏢,包含元大金(2885)、富邦金(2881)、凱基金(2883)、
國巨*(2327)、富邦媒(8454)、保瑞(6472)、藥華藥(6446)、信驊(5274)、潤弘(2597)等主流權值/中大型股。
還原價是**所有 NAV 回測與 live S 出場重放的地基**(canonical prices module),這些股票在除權日會多一筆
假跌損,污染那段報酬。

**已知但未修**:`src/quantlib/prices.py:314-315` 的註解白紙黑字寫「兩者皆無 → null…那些是 pre/ref/cash 全 0 的
壞列,須重解析 ex_right_dividend 補回,見 FC1-parse」——開發者已經知道這批 0 列是壞的、掛了 TODO,但還沒補。

**修法**:MOPS 檔既然有配股元/股,除權還原因子可由面額回推:`factor = 10 / (10 + 股票股利元/股)`
（台股面額 10 元）。要嘛在解析時把這個 factor 反算成一個合成的參考價/前收盤填進價格欄,要嘛在 schema
新增一欄 `stock_dividend_per_share` 讓 `prices.py` 直接吃。順便把配息 MOPS 列的現金也保留(現況已保留,OK)。

### SUSPECT 2 — tpex 新舊雙源同時匯入,靠唯一索引擋,解析誰贏靠運氣
tpex 在 2025-2026 **同時**存在舊制範圍檔(2026_1_21.csv,有真實價格)與 MOPS 月檔(2026_1.csv,價格全 0),
兩者涵蓋同一批除權息事件、`(market,date,company_code)` 唯一鍵相同,且**在同一次 `readExRightDividend()`
裡被平行(`.par`)一起處理**。誰的值最後入庫,取決於哪個檔先被平行工作插進去(去重閘 `dataAlreadyInDB` 是
迴圈開始前拍的快照,同一 run 內兩檔都會通過閘),最後靠唯一索引 `idx_ExRightDividend_market_date_companyCode`
擋掉後到者。**現況無害**:實測「PG 存 0 但同一事件在磁碟舊制檔有真實價格」= **0 筆**——舊制的真價格在
兩者並存處全部贏了。但這是**時序/平行順序相依**的僥倖:全庫清空重建時,MOPS 的 0 值可能非確定地贏過真價格,
或觸發一串 unique-violation 例外。屬健壯性技術債,值得順手記下。

### note 3 — MOPS 丟掉 211 筆純現金增資除權(與舊制同範圍,非退化)
MOPS 月檔含 211 筆(2024:68、2025:87、2026:56)「有除權交易日、但無配股也無配息」的純現金增資(現增)事件
(如光罩 2338、國喬 1312 現增 4 億股),解析器不發任何列。查證:舊制檔案**本來也幾乎不收**這類事件
(12 年大批量檔只 3 筆、2025/2026 舊制檔 0 筆)——所以丟掉它們是**維持與舊制一致的除權息表範圍**,
不是退化。若未來需要現增的除權還原,MOPS 檔有料可補,但那是新需求不是既有 bug。

### note 4 — `cash_dividend` 欄跨時代語意漂移
舊制此欄存的是「權值+息值」(交易所把配股也折算成價值加進去),MOPS 此欄存的是純現金(盈餘+法定公積+特別股現金)。
純配息事件兩者一致;配股配息混合事件,舊制的 `cash_dividend` 會比實際現金多(含權值)。**不影響 `prices.py`**
(舊制走參考價/前收盤,不吃這欄;MOPS 的現金 fallback 只對純配息列觸發,那時兩者相等)。純屬欄名與語意的
歷史包袱,記錄備查。

## 資料可信度總結

| 範圍 | 可信度 |
|---|---|
| twse 2003–2024/06、tpex 2008–（舊制檔覆蓋處） | ✅ 完全可信(逐格對帳 + 語意交叉驗證雙綠) |
| 所有時代的**現金股利(配息)** | ✅ 可信(值正確,下游還原正確) |
| 2024-07+ **純配股(除權)** | ❌ 整列 0,下游還原失效(59 檔,BUG 1) |
| 2024-07+ **現金增資除權** | ➖ 不收,但與舊制同範圍(note 3) |
