# 資料稽核 C-taifex_futures_daily — 期貨日檔 cache 一致性與缺漏

**結論(白話):** Cache 和 PostgreSQL **一模一樣、可以完全信任**——我逐欄、逐筆、
整日都對過,零差異。但**期貨表本身有兩段大洞,是根本沒抓進來**(不是 cache 出錯):
**2026 年 1-2 月整整兩個月**、以及**2026-05-22 到今天**都沒有資料,合計約 **72 個交易日**
查嘸。任何用到這兩段期間期貨資料的回測或訊號,會拿到空序列或被靜靜截斷。

**Verdict: BUG**(一致性本身 OK,但時間序列有真缺口 → 資料不完整)

---

## 1. Cache vs PG 一致性:完美(OK)

| 檢查 | 結果 |
|---|---|
| Schema | cache 19 欄 vs PG 20 欄。**唯一差異是 `id`(bigint 自增代理主鍵)被刻意捨棄**,不影響分析。19 個分析欄一個不漏、型別無降級(`cache_tables.py:64-68` 明列)。 |
| 整表筆數 | 兩邊都 **5,780,185**。 |
| 逐年筆數 | 1998-2026 共 29 年**逐年完全相同**。 |
| 日期範圍 | 皆 1998-07-21 ~ 2026-05-21;distinct 交易日 6877、契約 751、盤別 2,全同。 |
| 逐欄聚合 | 14 數值欄 + 4 字串欄的 count/sum/min/max/非空數/distinct,DuckDB 同時掛 PG(唯讀)與 cache 計算 → **0 mismatch**(sum_close 僅第 13 位有效數字差,浮點加總順序,可忽略)。 |
| 整日逐筆 | 對 2026-05-21 / 2020-06-15 / 2011-08-10 三天做雙向 `EXCEPT` 全欄比對 → **pg-only=0、ca-only=0**;另 3 日 × 5 契約(TX/MTX/TE/TF)顯式逐值 dump 全 match。 |
| 重複列 | cache 在唯一鍵 (date, contract_code, contract_month, trading_session) 上 distinct = total → **零重複**,符合 PG UNIQUE index。 |

**cache 同步程式忠實無誤,這張表 cache 端可放心使用。**

## 2. 時間序列缺口:兩段真缺口(BUG)

期貨月度覆蓋一看就懂:

```
2025-11  20 天   2025-12  22 天
2026-01   0 天 ← 缺   2026-02   0 天 ← 缺
2026-03  22 天   2026-04  20 天   2026-05  14 天(只到 05-21)← 停
```

### 缺口 1:2026-01-02 ~ 2026-02-26(33 個交易日)
期貨日期從 **2025-12-31 直接跳到 2026-03-02**。同期間 daily_quote(股票)有 33 個交易日、
`is_trading_day('2026-01-05')=True`,證明是真交易日。這是**被前後完整資料包夾的中間洞**,
不能用「還在回補」解釋。根因:原始檔 `data/taifex/futures_daily/2026/` 只有 `2026_3/4/5.csv`,
**沒有 `2026_1.csv`、`2026_2.csv`** → 從沒下載。

### 缺口 2:2026-05-22 ~ 2026-07-20(39 個交易日,還在擴大)
期貨最新只到 **2026-05-21**,但股票日報已到 **2026-07-20**。之後 39 個交易日期貨全無。
原始檔沒有 `2026_6.csv`/`2026_7.csv`,且 `2026_5.csv` 是半個月檔(只到 05-21)。
**期貨爬蟲在 5 月下旬後停擺,沒跟上每日更新。**

> 兩段缺口在 PG 和 cache 裡一模一樣 → **不是 cache 同步問題,是爬取層根本沒抓**。

**補法(交主流程執行,勿自行下載):** 官方免費端點
`https://www.taifex.com.tw/cht/3/futDataDown`(`application.conf:153`,down_type=1 抓當年
日期區間 CSV)可回補 2026-01/02 與 2026-05-22 起至今;補齊 `2026_5.csv` 下半月;
匯入 PG 後重建 cache。並建議把 taifex 日檔納入每日 `Main update`,避免再停擺。

### 非缺口(排除):2009-12-12
這天股票有值但期貨沒有,`is_trading_day('2009-12-12')=False`(週六)→ 期貨當天合法未開,
**不是漏抓**。這是全 22 年唯一的其他「洞」,已排除。

## 3. 異常值掃描:全數為 TAIFEX 真實結構(REAL,非 bug)

物理不可能項**全為 0**:high<low、close/open 超出當日高低(成交量>0)、負價、負量、
負未平倉、未來日期。以下「看似異常」查證後皆為真實市場結構:

- **63% 列全 OHLC 為 null(365 萬筆)**:其中 298 萬帶結算價、299 萬帶未平倉 →
  掛牌但當日沒成交的遠月契約,TAIFEX 每個掛牌契約月都出一列。**真實。**
- **成交量>0 但 close 為 null(28.7 萬筆)**:20.5 萬筆有價差量、契約月形如
  `202506/202507` → **月間價差(calendar spread)組合列**,本就沒有離群收盤價。**真實。**
- **買價>賣價(38 筆)**:全部在 1999 年 → 早期 TAIFEX 快照舊報價殘留。**真實、極少。**
- **收盤價 <1**:全屬 CDF/XAF/DHF 等**個股/匯率期貨**,價格本就小。**真實。**
- **change_pct 精確落在 ±10(1462 筆)/ ±20(4 筆)**:正是 TAIFEX 每日漲跌幅限制。**真實。**
