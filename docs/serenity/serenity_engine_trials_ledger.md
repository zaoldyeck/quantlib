## 戰役六:估值目標止盈 vs 固定 +60%(2026-07-06 預註冊;使用者提案)

**假設**:止盈價應由「預估成長 vs 估值」推導(PEG 邏輯),而非固定 +60%。
**變更**(固定參數,禁止事後調整;成長代理 = yoy_3m 營收動能,PE = stock_per_pbr PIT):
- A `ev_v2_tpdyn`:每檔動態止盈倍數 = clip(yoy_3m / PE, 1.2, 3.0)(即 PEG=1 隱含上檔,夾在 +20%~+200%);PE 或成長缺失/非正 → 回退 1.6。進場時設定,每次 refresh 更新。其餘規則同 v2。
- B `ev_v2_pegexit`:取消固定止盈,改為「估值變貴出場」——refresh 日檢查 PE / yoy_3m ≥ 1.5 且未實現 ≥ +20% → 出場。其餘規則同 v2。
**已知風險(先寫下)**:循環股在獲利高峰 PE 最低(宇瞻 9.6、創見 9.4),PEG 目標在週期頂點會叫你「續抱」——這是 PE 類指標用在 memory 股的經典陷阱;預期 A 的動態上限會在 memory 股上退化成「幾乎不止盈」,等同已被否決的 no-tp。
**判準**:三窗中 ≥2 窗 Sortino 改善且 MDD 均值惡化 ≤2pp 才採納。trials +2 → **46**。
**結果(2026-07-06)**:**雙雙否決(A 0/3、B 0/3)**。
- A tpdyn Sortino:4.479/4.082/3.575 vs v2 4.722/4.864/4.000(CAGR 216.5/177.0/146.9 vs 230.7/210.1/178.9)
- B pegexit Sortino:3.811/4.309/3.395;lag90 CAGR 222.1 較高但 MDD/Sortino 皆劣化,單窗噪音
- **失效機制(比預期更完整)**:PEG 目標產生反向選擇——高 PE 動能股(台光電 PE 110)的隱含目標被夾到 +20% 提早賣出最強持股;低 PE 循環股(南亞科 trailing PEG≈0.04)拿到 +200% 上限而在週期頂點續抱。循環股獲利高峰時 PE 最低=估值指標最樂觀的時刻恰是該離場的時刻(trailing 資料的結構性缺陷)。
- 邊界註記:本測試的成長代理為營收 yoy_3m、PE 為 trailing——若未來取得 forward EPS 共識資料可重測 forward-PEG 一次;DCF 依 2025 replay 證據定位為 entry-side overpay guardrail,不做出場工具。
- **估值在本系統的正確位置(維持現狀)**:進場計分(PE 分級扣分已內建)、排序(PEG IC 為正)、買貴防呆;**出場交給價格(trailing/止盈回收)與營收動能(論點停損)**。

## 戰役五:2020-2022 回溯策展(哲學泛化裁判;預註冊)

**方法**:僅用當期可考證的公開證據(貿易報導/法說時點)建三份歷史註冊表(2020 防疫/宅經濟、2021 航運+ABF/被動元件、2022 能源/軍工),候選股採產業分類機械成員(禁止手挑個股),全部跑 lag0/90/180。
**偏誤聲明**:策展者(LLM)知道歷史結局,無法完全去偏;證據時點紀律 + lag 壓力 + 機械成員資格為三道緩解。結論只作方向性判讀。
**判準**:三個年代註冊表在 lag90 下仍全數顯著勝 0050 → 哲學泛化成立;任一年代失效 → 系統定位改為「單一超級週期策略 + regime kill-switch」。
**結果(2026-07-06)**:**有條件成立**(嚴格判準未全過,但方向性證據強):
- 合併 registry 2020-01~2023-12(9 個非 AI 主題,類別成員+證據日期):ev_full_tp60_v2 → lag0 26.5%/-21.5%、lag90 18.6%/-26.3%、lag180 19.2%/-24.1%,三個 lag 全勝 0050(12.4%/-34.0%)於 CAGR/Sharpe/MDD
- 分年:**2021 大勝**(超額 +60.4/+70.4/+96.5pp @lag0/90/180——lag 越大超額越大,排除啟用日作弊);2022 空頭同跌略淺;2023 持平;**2020 在 lag90/180 失效**(-24.7/-33.3pp,covid/宅經濟窗口太短,延後啟用即錯過)
- 判讀:**長週期瓶頸主題(貨櫃/ABF/成熟製程,供給無彈性 ≥18 個月)可被同一引擎跨 regime 放大;短窗口事件型主題(<12 個月)不適用**。covid_supply 事後檢視本就違反 curation SOP 準入第 2 條(口罩產能 6 個月擴出)——SOP 準入標準獲實戰校驗,已修訂加入「瓶頸存續預期 ≥12 個月」
- 對照:rotation_only(無出場/guard)lag0 46.5% 但 MDD -58.6%,lag180 崩至 14.2% 落後 2330——**紀律層是哲學可攜性的必要條件**(2025 的 alpha 主體在策展,跨週期的存活主體在紀律)
- 量級校準:常規週期的合理期望 = 勝 0050 約 6-14pp/年 + 更淺 MDD;2025 的 230%+ 是 AI 超級週期特例,不可外推

## 池天花板測量 — Serenity 加引擎有無機會勝過 Evergreen(2026-07-16 預註冊;使用者提問「跑引擎優化之前有沒有辦法先測」)

**方法(零 token 純計算;月頻框架,對兩池同等保守)**:引擎 = 池內排序+擇時,天花板
用兩把尺、兩池同量:
1. **Oracle top-K(先知上界)**:每月以「當月實現報酬」事後挑池內最強 K 檔等權——任何
   引擎在月頻框架的數學上限;不可實現,量的是「池內可被排序收割的分散度」。
2. **Momentum top-K(現成排序代理)**:每月以「前月末 60 交易日報酬」(PIT)挑 top-K——
   零優化、零新發明,量「搬現成訊號能實現多少」。
K ∈ {5, 10}(Evergreen 現役 5 席、Serenity 現役 10 席);另量兩池等權月報酬相關性與
50/50 blend(互補性)。
**判準(預先定義,以 CAGR 為準、bootstrap P5 佐證)**:
- **無望**:Serenity oracle top-10 ≤ Evergreen 等權 → 池內好料不足,引擎再強也追不上,
  省下引擎優化投資;
- **有據**:Serenity momentum top-10 ≥ Evergreen 等權 → 現成排序即可追平,引擎優化
  (含已驗證的八成分計分)大概率更高,投資有據;
- **中間帶**:oracle 夠高但 momentum 追不上 → 上限存在、排序難度高,引擎投資屬
  高風險高報酬,交使用者裁決;
- 互補性獨立報告:若兩池月報酬相關 < 0.7 且 blend 改善 MDD/Sortino → 「即使不勝,
  配置價值成立」。
**誠實聲明**:oracle 受池大小影響(Serenity 池均 28.7 檔選項多、天然佔 oracle 便宜)——
故同時報 oracle 溢價(oracle − 等權)與兩池同 K 對比;月頻框架量不到日頻擇時/出場的
上限,對兩邊同等保守。
**結果(2026-07-16)**:**中間帶——上限夠高,現成排序追不上;但近段實證顯示現役引擎

  屬高風險高報酬)。
- **判準外實證(關鍵)**:近段(2025-01~2026-07)Serenity 等權僅 113.1%,而現役引擎

  已勝過 Evergreen 裸池**。開放問題只剩一個:此捕捉能力在 2022-24 非 AI regime 是否
  成立——戰役十七(出場參數弱敏感)已預示需要 regime 自適應,walk-forward 正是為此。
- **互補性**:兩池等權月報酬相關 0.66,50/50 blend CAGR 140.2%(介於兩者,MDD 無明顯
  改善)——配置分散價值中等。
