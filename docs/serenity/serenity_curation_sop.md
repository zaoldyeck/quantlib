# Serenity 策展 SOP — 論點註冊表的生產與維護

回測已證明(serenity_event_engine_v1.md):**alpha 的主體在策展層**(池內隨機選股仍有
~130-140% CAGR,機械主題偵測則無 alpha)。因此註冊表不是設定檔,是策略的核心資產,
必須像生產線一樣維護。本 SOP 定義準入、證據、失效與節奏。哲學來源見
`.claude/skills/serenity-trading-system/`(Chokepoint Theory)。

## 1. 準入標準:瓶頸簽名(theme-agnostic)

一個主題要進註冊表,必須同時滿足四項——**不限產業,AI 只是當前的解,不是定義**:

1. **需求跳升**:下游 TAM 有量級擴張的可信路徑(資本支出週期、政策、技術架構轉換)。
2. **供給無彈性**:上游擴產交期 ≥ 18 個月(產能、認證、材料、良率任一),且**瓶頸存續
   預期 ≥ 12 個月**——短窗口事件型缺口(如 2020 口罩,6 個月擴產補齊)經 2020-2022
   回溯實測對本引擎無效(lag 敏感度過高),一律拒絕。
3. **供應商集中**:節點由少數公司控制,且無 12 個月內可認證的替代者。
4. **放大器**(至少一項):出口管制/地緣、onshoring 政策、規格升級強制換代、
   漲價公告已出現。

反例(直接拒絕):純大宗商品循環(無認證鎖定)、下游品牌受益者(非瓶頸)、
已被媒體循環 frontrun 的主題(檢查 days-since-mainstream-coverage)。

## 1.5 成員層準入:瓶頸角色三測試(2026-07-14,戰役十五回測背書)

主題過瓶頸簽名後,**每個候選成員還要逐檔過三測試**——§1 反例的「下游受益者拒絕」
在成員層同樣執行(歷史教訓:memory_cycle 主題為真,但模組廠成員以「槓桿載具/
coverage gap」邏輯入冊,7-09 報告判斷層已書面認定威剛「不是瓶頸」仍居補位順位 1):

1. **稀缺節點持有測試**:這家公司「自己擁有」什麼稀缺物(產能/認證/專利/藥證/
   獨家規格)?答不出來 = beneficiary。
2. **定價權測試**:上游漲價/需求跳升時,它的毛利率是結構性擴張(報價-成本剪刀差
   走闊),還是一次性庫存財/成本轉嫁?
3. **替代性測試**:它的客戶 12 個月內能否無痛換掉它(認證週期、switching cost)?

**角色分類**(記錄於 `research/serenity/registry/member_roles.csv`,註冊表姊妹檔,
新成員入冊必標;role 是時不變結構屬性,以「該成員在該主題瓶頸節點的位置」為判準):
- `chokepoint_owner`:自有稀缺節點,漲價直接灌進自有結構毛利 → 入冊
- `enabler`:寡占關鍵投入/服務,有壁壘但非獨佔、或稀缺主體部分屬於他人 → 入冊
- `beneficiary`:瓶頸下游受益轉導(模組庫存財/組裝 ODM/純品牌通路)→ **不入冊**

回測背書(戰役十五,2026-07-14):排除 beneficiary 三窗 CAGR 全升(lag0 +63.0pp、
lag90 +31.7pp、lag180 +4.9pp),歸因梯度單調(owner 勝率 62-69% > enabler 54-62% >
beneficiary 41-56%);beneficiary 席位不是虧錢而是機會成本。**owner_only 亦被否決**
(enabler 是正貢獻,lag0 −52pp)——純度的最優點是「排除 beneficiary」,不是「只留
owner」,不得矯枉過正。

**龍頭條款**:同一節點多家過檢時,取規格/市佔領先者,**廢除 coverage-gap 完備化**
(「池裡已有同類所以補齊類別」是反向邏輯——發現池內有 beneficiary 時正確動作是
檢討出池,2026-07-07 sweep 為反例)。

**conviction 語義(新成員適用)**:conviction := 瓶頸強度——5 = 獨佔/近獨佔本尊、
4 = 寡占本尊、3 = enabler;不再混入「品質好/流動性夠」。既有成員 conviction
grandfather(全面重打改變計分輸入,需另開戰役驗證後才動)。

## 2. 證據要求(每個主題入表時必填)

| 欄位 | 要求 |
|---|---|
| `evidence_date` | 最早可公開查證的證據日期(貿易報導/法說/漲價公告)——**active_from 不得早於它** |
| `evidence_url` | 至少一條可回查的來源連結或文獻指引 |
| `invalidation_criteria` | 主題死亡的可觀察條件(見 §4)——入表時就寫,不許事後補 |
| `review_by` | 下次強制複審日(預設:入表 +2 個季度) |
| 候選股 | 以類別界定候選範圍(產業分類/產品線),**逐檔過 §1.5 成員層三測試 + 標注 role**;「禁止挑會漲的個股」= 禁止用近期報酬選成員,但結構角色判斷是義務、不是 cherry-picking;過檢者的量化優劣才交給引擎計分 |

## 3. 節奏與流程

- **每日**(可排程 agent 化,整合 `quantlib-daily-briefing`):跑 skill 的 L0-L4 資訊掃描
  (含非 AI 觸發源:SMM 全金屬價格、各產業漲價公告、航運運價、藥品/化工短缺通報、
  任何產業的出口管制)→ 產出候選主題與既有主題健康度註記,append 到 watch log。
- **動能發掘漏斗(2026-07-08 新增;2026-07-09 升級為引擎同源)**:
  `uv run --project research python research/serenity/discovery_funnel.py` ——
  用**引擎一模一樣的動能子分數**(ret_60d/20d/252d 同 clip 同權重)+ 同源硬門檻掃全市場、
  排除已在池者,取 top-N 作為**發掘輸入,非買訊**——動能本身無 alpha(置換檢定:機械篩
  選 0 alpha),但高動能常是「市場開始定價一個我們還沒策展的瓶頸」的訊號;用引擎同一把尺
  取代手刻漲幅榜,血統一致。**過濾一律按結構(瓶頸簽名),
  絕不按產業別**(theme-agnostic 鐵律):唯一自動排除的是**結構上不可能持有瓶頸的角色**——
  純通路商/貿易商(中間人不持有稀缺節點)、無差異化組裝廠、純控股/金融。其餘一律送 §1
  瓶頸簽名檢核由結構判定,**不因產業別預先淘汰**——生技(法規/專利壟斷,如已在冊的藥華藥
  biotech_regulatory)、特化材料、汽車零件、機能紡織、化工特用品…都可能是 sole-source 瓶頸。過檢者才進註冊表,不插隊(下次 refresh
  自然入池)。與營收拐點漏斗(emerging-leader-scan)並列為兩條發掘管道。**注意**:動能
  已被引擎計分內建(ret_60d/252d),所以本漏斗的獨特價值是找「尚未在冊」的瓶頸,補策展缺口。
- **每月**(營收公布後):對照引擎 refresh 的池品質;主題內成員營收廣度惡化 → 提前複審。
- **每季**:全表強制複審;過了 `review_by` 未複審的主題自動降 conviction 一級。
- 新主題入表、舊主題失效**都要 git commit**(留審計軌跡,這是前瞻證據的一部分)。

## 4. 主題失效(theme-level thesis stop)

滿足自身 `invalidation_criteria` 或以下任一通用條件 → 設定 `active_until`,引擎自然停止進場:

- 瓶頸解除:供給端擴產到位/替代者通過認證/交期恢復正常。
- 需求端斷裂:下游資本支出指引下修 ≥ 2 家主要買家。
- 價格訊號:該節點產品報價連續 2 季回落。

**Regime kill-switch(引擎 v2 已代碼化)處理的是「市場對主題的定價」;本節處理的是
「主題本身的事實」——兩者獨立運作。**

## 5. 帳本與紀律

任何影響引擎行為的變更(計分、出場、guard 參數)walk through
`serenity_engine_trials_ledger.md`:先預註冊假設與判準,再跑,如實記錄。
註冊表內容變更不需要 ledger(它是策展,不是參數),但需要 git commit + 證據欄位齊全。
